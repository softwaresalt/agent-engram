## Key Decisions

1. **Windows named pipe readiness**: `std::fs::metadata()` cannot detect if a named pipe server
   is listening. Fixed by replacing filesystem probe with `check_health()` IPC call in DaemonHarness.
2. **Canonical path hash alignment**: Windows `canonicalize()` adds `\\?\` prefix. Both daemon
   and harness must use `std::fs::canonicalize()` on the same path for SHA-256 DB namespace to match.
3. **rmcp 0.13.0 API** (Cargo.toml says "1.1" but resolves to 0.13.0):
   - `rmcp::serve_server(handler, transport).await` → `RunningService<RoleServer, S>`
   - `rmcp::transport::io::stdio()` → `(tokio::io::Stdin, tokio::io::Stdout)` as transport
   - `running.waiting().await` → `Result` (must be propagated, not discarded)
4. **Timing threshold**: spec SLA is 2s but debug builds take 5–10s; test threshold set to 10s.
5. **F-4 list_tools deferred**: `list_tools` returns empty list from shim — acceptable for Phase 3.
   Tool discovery proxying to daemon deferred to a future phase.
