## Key Decisions

1. SurrealDB `.bind()` requires owned values — `.clone()` / `.to_owned()` on all string bindings to avoid E0521 lifetime errors.
2. CodeGraphQueries is a separate struct from Queries — both wrap `Db` but handle different table families.
3. tree-sitter Parser is `!Send` — must be created inside `spawn_blocking` for async use.
4. proptest nested tuples for >12 fields — Function has 13 fields, split into two sub-tuples.
5. Clippy `similar_names` allowed on edge creation methods (caller_id/callee_id is domain language).

## Failed Approaches

- Using `&str` references in SurrealDB `.bind()` — causes E0521 lifetime errors. Must use owned values.
- Using `ref` in match patterns with Rust 2024 edition — disallowed when default binding mode is not `move`.
- Single proptest tuple with 13 elements — max supported is 12, must nest sub-tuples.
