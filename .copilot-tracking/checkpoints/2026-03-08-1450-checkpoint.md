## Key Decisions

1. **Feature-gate over removal** (`legacy-sse`): Chose a Cargo feature flag over outright deletion to preserve HTTP/SSE code for potential future re-enablement while eliminating the dead-code lint violation (ADR-0016).
2. **Skip for deletions/renames in `adapt_event`**: Deleted/renamed files map to `ServiceAction::Skip` because targeted per-file graph deletion would leave stale edges; full `sync_workspace` is the correct reconciliation path.
3. **108-byte Unix socket limit**: Conservative cross-platform UNIX_PATH_MAX (Linux=108, macOS=104). macOS paths of 105–108 bytes would still fail — acceptable tradeoff documented in code.
4. **Workspace path (not sock path) for fallback hash**: The fallback `/tmp/engram-{hash}.sock` uses the workspace path as hash input (not the socket path) to match the Windows naming convention and ensure determinism.
5. **`is_some_and` vs `map_or`**: Used `is_some_and` in `is_code_file` to satisfy Clippy's `unnecessary_map_or` pedantic lint (Rust edition 2024 pattern).
