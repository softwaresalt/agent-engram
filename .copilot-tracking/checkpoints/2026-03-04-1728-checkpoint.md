## Key Decisions

1. **Box::leak for fd-lock guard lifetime**: `fd_lock::RwLock::try_write()` guard lifetime is bounded by the lock. Used `Box::leak(Box::new(RwLock::new(file)))` to get `'static mut` reference so guard can be stored in `DaemonLock` struct. This is safe (no unsafe blocks) and acceptable since daemon holds exactly one lock for its lifetime.

2. **IpcError name disambiguation**: `crate::errors::IpcError` = domain enum. `crate::daemon::protocol::IpcError` = wire-format struct. Both coexist; `ipc_server.rs` imports them as `WireError` and `DomainIpcError` respectively.

3. **Windows pipe naming**: `to_ns_name::<GenericNamespaced>()` requires pipe name WITHOUT `\\.\pipe\` prefix. Strip prefix before calling API: `endpoint.strip_prefix(r"\\.\pipe\").unwrap_or(endpoint)`.

4. **_shutdown deferral (T052)**: `_shutdown` IPC method returns correct contract response but does NOT halt the accept_loop. Loop only breaks on Ctrl-C. Full shutdown via `CancellationToken` deferred to T052 (Phase 8).

5. **PID file truncation bug fix**: After acquiring lock, call `guard.set_len(0)?` before `seek(0)` + `write_all()` to prevent stale bytes from longer previous PID surviving.

6. **Stale socket removal**: Changed `let _ = remove_file()` to propagate non-NotFound errors, eliminating silent failure diagnostic loss.

7. **read_line size cap**: Added `const MAX_REQUEST_BYTES: usize = 1024 * 1024` with match arm `Ok(n) if n > MAX_REQUEST_BYTES` returning parse error.
