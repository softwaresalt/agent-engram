## Key Decisions

1. **notify-debouncer-full 0.7.0 pins to notify 8.x internally** — despite notify 9.0.0-rc.2 in
   Cargo.toml, the debouncer uses notify 8.x. Use `debouncer.watch()` directly (not `.watcher().watch()`).
2. **Degraded mode**: `start_watcher()` returns `Ok(None)` on debouncer init failure so daemon
   continues without file watching (vs. hard error).
3. **Windows exclusion**: `is_excluded()` matches both the directory name itself AND all descendants
   because Windows `ReadDirectoryChangesW` fires `Modified` on parent directory when child is created.
4. **Rename event**: `paths[0]` = old path, `paths[1]` = new path for `RenameMode::Both`.
5. **debounce.rs is thin in Phase 4**: actual debounce logic lives in `watcher.rs` via debouncer.
