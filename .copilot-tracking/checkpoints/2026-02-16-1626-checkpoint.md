### Key Schema from contracts/mcp-tools.json
- **map_code input**: `symbol_name` (required), `depth` (default 1, max 5), `max_nodes` (default 50, max 200)
- **map_code output**: `root` (CodeNode|null), `neighbors` ([CodeNode]), `edges` ([Edge]), `truncated`, `fallback_used`, `matches` (for multi-match), `effective_depth`, `effective_max_nodes`
- **list_symbols input**: `file_path?`, `node_type?` (function/class/interface), `name_prefix?`, `limit` (default 50, max 500), `offset` (default 0)
- **list_symbols output**: `symbols` ([{name, type, file_path, line_start, line_end}]), `total_count`, `has_more`
- **CodeNode**: `{id, type, name, file_path, line_start, line_end, signature, body, embed_type, summary}`
- **Edge**: `{type, from, to}` — type is calls|imports|inherits_from|defines|concerns

### Queries Needed (T038)
- Multi-table symbol lookup: search function + class + interface by name
- BFS 1-hop: get all edges (calls/imports/inherits_from/defines) incident to a node
- BFS multi-hop: iterative 1-hop expansion with visited set and max_nodes cap
- Symbol listing: filter by file_path, node_type, name_prefix with LIMIT/OFFSET
- Symbol count: total matching count for pagination

### Existing query patterns in queries.rs
- All queries use `self.db.query("...")` with `.bind()` for parameters
- Return types use internal Row structs (FunctionRow, ClassRow, InterfaceRow) with `into_function()` conversions
- Table `function` MUST be backtick-escaped in raw SurrealQL
