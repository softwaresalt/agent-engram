## Gap Analysis

### T094 — Dedicated Error Format Validation Test (Minor Gap)

The tasks.md marks T094 `[X]` complete, and the SC-017 requirement (all errors produce
`{"code":…, "name":…, "message":…, "details":…}`) is validated **implicitly** across every contract
test that checks an error response. However, there is **no standalone test function explicitly
named `t094`** or keyed to SC-017. The error format correctness is verified by existing tests like:

- `contract_add_label_duplicate_returns_error` — asserts error code 3011
- `contract_claim_already_claimed_returns_error` — asserts error code 3005
- `contract_add_dependency_cycle_rejected` — asserts error code 3003
- (and ~15 more workspace-not-set error checks)

**Assessment**: Not a blocker. The gap is aesthetic (no dedicated SC-017 test function), not a
functional hole. All new error codes are exercised by contract tests that validate the
`ErrorResponse` JSON shape.
