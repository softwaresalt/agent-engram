## Key Decisions

1. **rmcp 1.1 over mcp-sdk 0.0.3**: rmcp is the official Rust MCP SDK with active maintenance, StdioTransport support, and proper ServerHandler trait. mcp-sdk had zero active development (ADR 0015).
2. **notify-debouncer-full 0.7.0 upgrade**: Upgraded from 0.5.0 (the version originally pinned in tasks.md) to 0.7.0. However, 0.7.0 still depends on notify 8.2.0 not 9.x — both versions are in the dep tree during Phase 1.
3. **Defer notify version decision to Phase 4**: When the watcher is implemented in Phase 4, must choose between (a) downgrade notify to 8.x + keep debouncer or (b) keep notify 9.x + implement manual debouncing.
4. **`#[allow(clippy::unused_async)]` on all stubs**: Required because async functions with only `todo!()` trigger pedantic lint. Remove when real implementations land.
5. **`reason` field in `details`**: match arms for IpcError::ConnectionFailed, LockError::AcquisitionFailed, and WatcherError::InitFailed include `path`/`address` in details but not `reason` (reason is in the error message string). This was deliberate — reason is in message, details carries structured data for programmatic handling.

## Failed Approaches

- **Multiple concurrent cargo processes**: Running many cargo builds simultaneously caused build directory file lock contention on Windows. Resolved by killing all cargo processes and running a single clean test.
