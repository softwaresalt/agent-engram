## Key Decisions

1. **No set_workspace IPC call needed** — The daemon auto-binds the workspace from `--workspace` CLI arg on startup. Calling `set_workspace` via IPC hits the workspace limit (code 1005). Use `get_workspace_status` to verify binding instead.
2. **Pass `Some(json!({}))` not `None` for tool params** — Tools doing `serde_json::from_value(params.unwrap_or_default())` fail when params is None (Value::Null doesn't deserialize as a struct). Empty object works because all fields have `#[serde(default)]`.
3. **`create_task` returns `task_id` not `id`** — The response JSON uses `task_id` as the key. The `id` field is used in serialized task objects (via `serialize_task` in output.rs) in list responses like `get_ready_work`.
4. **Windows IPC readiness** — Named pipe readiness cannot be detected via `Test-Path` reliably; the run-local script uses a polling loop checking pipe existence, while tests use `check_health()` IPC ping.

## Failed Approaches

- **`set_workspace` in smoke test Step 1** — Returned error code 1005 (WORKSPACE_LIMIT_REACHED) because the daemon already holds the workspace slot from startup. Replaced with `get_workspace_status`.
- **`None` params for `get_ready_work`** — Failed with InvalidParams because `Value::Null` can't be deserialized as a struct. Fixed by passing `Some(json!({}))`.
- **`task_result["id"]` assertion** — Wrong field name; `create_task` returns `task_id`. Fixed after reading the `write.rs` source.
